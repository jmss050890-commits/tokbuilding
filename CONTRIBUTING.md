# CONTRIBUTING.md

## Contributing to SVL-KPA

Thank you for your interest in contributing to the SVL-KPA platform by Sanders Viopro Labs LLC. Our mission is to Keep People Alive (KPA) through innovative, secure, and compliant technology.

### Guidelines
- All contributions must align with the KPA mission and SVL's legal guardrails.
- Document all design and code changes in DESIGN_HISTORY.md and AUDIT_TRAIL.md.
- Add new dependencies to LICENSES.md with license details.
- Follow our code style and branding standards (see svlColors.ts).
- Do not submit proprietary or third-party code/assets without proper licensing and documentation.

### How to Contribute
1. Fork the repository and create a feature branch.
2. Make your changes with clear, descriptive commits.
3. Update relevant documentation files.
4. Submit a pull request with a summary of your changes and their purpose.
